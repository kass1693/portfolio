# Key Fetcher

AP 관리 프로토콜 암호화 과정에서 필요한 키 교환 시스템입니다.  
예외 상황이 많고 스펙 변경이 반복되는 환경에서, 전체 로직을 다시 고치지 않고 영향이 있는 지점만 수정할 수 있도록 흐름을 나누는 것을 목표로 설계·구현했습니다.

## 핵심 내용

- 재시도, 타임아웃, 상태 전이, 로그 처리를 분리해 스펙 변경 시 수정 범위를 줄였습니다.
- 반복된 변경 요청에도 전체 흐름 대신 영향이 있는 정책과 단계만 수정하는 방식으로 대응했습니다.
- 운영 중 문제 발생 시 상태 흐름과 로그를 기준으로 원인을 좁힐 수 있도록 정리했습니다.

## 결과

- 스펙 변경 16건 대응
- 평균 1일 내 수정 반영
- 6개월 이상 필드 테스트 운영

## 제 역할

요구사항 분석부터 설계, 구현, 테스트, 운영 대응까지 전체 흐름을 맡았습니다.  
핵심은 기능을 추가하는 것보다, 변경이 반복될 때 수정 지점을 분리하고 문제를 추적할 수 있도록 기준을 세우는 일이었습니다.
