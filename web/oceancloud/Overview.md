# Overview

## 프로젝트 개요

Ocean Cloud는 자료를 수집하고, 저장된 데이터를 다양한 방식으로 조회할 수 있도록 만든 웹 플랫폼입니다.  
이 프로젝트의 중심은 단일 화면이나 단일 기능이 아니라, 입력과 조회, 시각화가 하나의 흐름 안에서 이어지도록 만드는 일이었습니다.

사용자는 화면에서 자료를 등록하고, 저장된 데이터를 조건에 따라 검색하고, 필요하면 지도나 차트 형태로 확인하고, 다시 내려받아 활용할 수 있어야 했습니다.  
그래서 이 프로젝트는 단순한 입력 화면 개발보다, 자료를 다루는 전체 흐름을 웹에서 끊기지 않게 연결하는 데 더 가까운 프로젝트였습니다.

## 어떤 문제를 풀었는가

처음부터 고정된 입력 폼과 단순 조회 화면만으로는 요구사항을 감당하기 어려웠습니다.  
입력 항목은 계속 바뀔 수 있었고, 조회 방식도 단일 테이블 수준이 아니라 조건 검색, 지도 기반 확인, 차트 시각화, CSV Export처럼 여러 방향으로 확장되어야 했습니다.

이런 상황에서 화면 하나를 기능별로 따로 만드는 방식으로 접근하면, 입력 항목이 바뀔 때마다 프론트엔드 수정 범위가 커지고, 조회 기능이 늘어날수록 상태 관리와 화면 흐름도 복잡해질 수 있었습니다.  
그래서 Ocean Cloud에서는 자료를 등록하고 조회하는 전체 흐름을 먼저 보고, 화면과 API, 데이터 저장 방식을 그 흐름에 맞게 정리하는 쪽으로 접근했습니다.

## 주요 기능

### 1. 서버 정의 기반 동적 입력

입력 화면은 고정된 폼을 화면마다 따로 만드는 방식이 아니라, 서버에서 내려주는 정의를 기준으로 동적으로 구성되도록 구현했습니다.  
이 방식의 목적은 입력 항목이 바뀔 때마다 프론트엔드 화면 전체를 다시 수정하지 않도록 하는 데 있었습니다.

이를 통해 다음과 같은 효과를 얻을 수 있었습니다.

- 입력 항목 변경 시 화면 수정 범위 축소
- 공통 입력 흐름 재사용 가능
- 새로운 입력 유형 추가 시 대응 속도 향상

### 2. 데이터 탐색 기능

자료를 저장하는 것만으로는 충분하지 않았기 때문에, 저장된 데이터를 다양한 조건으로 조회할 수 있는 탐색 기능을 함께 구현했습니다.  
사용자는 필터 조건, 검색 조건, 화면 상태를 기준으로 필요한 데이터만 빠르게 좁혀 볼 수 있어야 했습니다.

이 기능에서 중요했던 점은 단순 조회 자체보다, 여러 조건을 바꿔가며 탐색하는 과정이 자연스럽게 이어지도록 만드는 것이었습니다.

### 3. 지도와 차트 기반 시각화

Ocean Cloud에서는 데이터를 단순 표 형태로만 보여주지 않고, 성격에 따라 지도와 차트로도 확인할 수 있도록 구성했습니다.

- 지도 기반 위치 확인
- Marker Clustering
- 차트 기반 데이터 요약
- 화면 상태에 따른 시각화 전환

이 기능들은 보기 좋은 화면을 만드는 것보다, 자료를 더 빠르게 이해하고 비교할 수 있게 하는 데 목적이 있었습니다.

### 4. CSV Export 및 부가 기능

탐색한 결과를 화면에서만 보는 것이 아니라, CSV Export를 통해 외부 활용이 가능하도록 구현했습니다.  
또한 Geolocation API를 활용한 위치 기반 기능도 함께 포함해, 실제 사용자 흐름 안에서 필요한 부가 기능을 연결했습니다.

즉, Ocean Cloud는 단순 입력/조회 화면이 아니라, 등록부터 확인, 시각화, 내보내기까지 이어지는 플랫폼으로 동작하도록 만든 프로젝트였습니다.

## 제가 맡은 범위

이 프로젝트는 책임급 개발자 1명과 2인으로 진행했습니다.
프론트엔드 화면 구현에만 머무르지 않고, 웹 서비스 전체 흐름을 함께 다뤘습니다.

주요 범위는 다음과 같습니다.

- React 기반 화면 구현
- 서버 정의 기반 동적 입력 시스템 구현
- 데이터 탐색 화면 구현
- 상태 관리와 화면 흐름 정리
- Express 기반 API 설계 및 구현
- MongoDB 연동
- 지도, 차트, CSV Export 같은 부가 기능 연결

즉, 특정 화면 하나를 담당한 것이 아니라, 자료를 입력하고 저장하고 다시 조회하는 전체 흐름을 웹에서 동작하게 만드는 역할을 맡았습니다.

## 사용 기술

이 프로젝트에서는 다음 기술을 활용했습니다.

| 영역 | 기술 |
|---|---|
| 프론트엔드 | React (Hooks), Context API, HOC 패턴 |
| 백엔드 | Express |
| 데이터 저장 | MongoDB |
| 시각화 | Google Maps API, Marker Clustering, Recharts |
| 상태 관리 | Context API, URL State Synchronization |
| 기타 | Geolocation API, CSV Export |

중요했던 것은 기술 자체보다, 각 기술을 입력, 조회, 시각화 흐름 안에서 어떻게 연결할 것인가였습니다.

## 이 프로젝트에서 중요했던 점

Ocean Cloud를 통해 제가 가장 많이 다룬 것은 웹 서비스에서 기능이 늘어날수록 화면, 상태, API, 데이터 저장 방식이 함께 복잡해진다는 점이었습니다.  
그래서 이 프로젝트에서는 기능을 개별적으로 추가하는 것보다, 입력과 탐색, 시각화가 서로 충돌하지 않고 하나의 흐름 안에서 이어지도록 정리하는 일이 중요했습니다.

정리하면 Ocean Cloud의 핵심은 다음과 같습니다.

- 자료를 등록하고 조회하는 웹 플랫폼을 구축한 점
- 동적 입력과 탐색 기능을 하나의 흐름으로 연결한 점
- 시각화, Export, 위치 기반 기능까지 함께 확장한 점
- 프론트엔드와 백엔드 전반을 함께 다룬 웹 프로젝트라는 점
